# ID 22

| Test Case Title/Scenario | Preconditions | Test Steps | Expected Results | Environment Specification |
|--------------------------|---------------|------------|------------------|----------------------------|
| SQL Injection (SQLi) Prevention | User is logged in; access to a search field or form that queries a database. | 1. Navigate to the search bar.<br>2. Enter a malicious string: ' OR '1'='1.<br>3. Submit the form. | The application should return no results or an error message. It must not return all records from the database. | Web Browser (Chrome/Firefox), Staging Environment, SQL Database. |
| Reflected XSS via Input Fields | Application has a user profile or comment section that reflects input back to the UI. | 1. Go to the "Edit Profile" page.<br>2. In the "Name" field, enter: <script>alert('XSS')</script>.<br>3. Save and view the profile. | The script should not execute. The input should be sanitized/encoded and displayed as plain text or blocked by a validation error. | Web Browser, Staging Environment, Sandbox User Account. |
| Malicious File Upload (Remote Code Execution) | Application has a "File Upload" feature (e.g., for profile pictures). | 1. Prepare a file named shell.php containing a simple script.<br>2. Attempt to upload this file as a profile picture. | The system should reject the file based on extension or MIME type. The file must not be saved to the server. | Web Browser, Staging Environment, Server with PHP/Backend support. |
| Input Length & Boundary Testing | A field has a defined character limit (e.g., 50 chars for a Username). | 1. Use a tool to bypass UI-level character limits.<br>2. Submit a string of 10,000 "A" characters via an API request. | The backend should validate the length and return a 400 Bad Request or similar error, preventing a Buffer Overflow or DoS. | Postman or Burp Suite, Staging API Endpoint. |
| HTML Injection in Email Notifications | System triggers automated emails based on user input (e.g., "Welcome [Name]"). | 1. Register a new account with the name <h1>Hacked</h1>.<br>2. Trigger the welcome email.<br>3. Open the email in a client. | The email should display the tag as literal text. The "Hacked" text should not appear as a large header. | Email Client (Outlook/Gmail), SMTP Test Server. |
