| Test Case Title/Scenario | Preconditions | Test Steps | Expected Results | Environment Specification |
|--------------------------|---------------|------------|------------------|----------------------------|
| TC-01: Brute Force Login Detection | 1. User has an active account.<br>2. Monitoring system is enabled. | 1. Navigate to the login page.<br>2. Enter correct username but incorrect password 5 times in 60 seconds. | 1. System blocks the IP or locks the account temporarily.<br>2. An alert (Email/SMS) is sent to the customer immediately. | Staging / QA Environment with SMTP Server enabled. |
| TC-02: Unauthorized Data Export | 1. User is logged in with "Standard" permissions.<br>2. A sensitive data set exists. | 1. Attempt to access a restricted data API endpoint directly via URL.<br>2. Attempt to download a bulk data report. | 1. Access is denied (403 Forbidden).<br>2. The unauthorized attempt is logged in the Security Dashboard.<br>3. Admin receives a high-priority alert. | Staging / QA Environment; API testing tool (e.g., Postman). |
| TC-03: Geographic Anomaly Detection | 1. User is currently logged in from New York.<br>2. VPN/Proxy tool is available. | 1. Attempt a login with valid credentials from a significantly different IP/Location (e.g., London) within 5 minutes. | 1. System flags the "Impossible Travel" scenario.<br>2. System challenges the user with MFA.<br>3. Security alert is triggered for the customer. | Production-like Staging with Geo-IP tracking enabled. |
| TC-04: Unauthorized Configuration Changes | 1. User is an "Admin".<br>2. Alerting for "Critical Changes" is on. | 1. Navigate to Security Settings.<br>2. Attempt to disable MFA or change the "Notification Email" for the account. | 1. System requires a password re-entry or MFA token for the change.<br>2. Notification is sent to the original email address alerting them of the change attempt. | Staging Environment; MFA simulator enabled. |
| TC-05: SQL Injection / Malicious Input Alert | 1. Application is live.<br>2. Web Application Firewall (WAF) is active. | 1. In a search or data field, input a common SQL injection string (e.g., ' OR '1'='1).<br>2. Submit the form. | 1. Application sanitizes input or rejects the request.<br>2. Security system logs the "Malicious Input" attempt and alerts the system admin/customer. | Staging Environment with WAF logging active. |
