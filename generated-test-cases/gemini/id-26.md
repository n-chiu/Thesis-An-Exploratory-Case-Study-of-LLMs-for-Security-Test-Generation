# ID 26

| Test Case Title/Scenario | Preconditions | Test Steps | Expected Results | Environment Specification |
|--------------------------|---------------|------------|------------------|----------------------------|
| Verification of Just-in-Time Permission Requests | App is freshly installed; no permissions granted yet. | 1. Launch the app.<br>2. Navigate to a feature requiring specific access (e.g., Camera for profile picture).<br>3. Observe when the permission prompt appears. | The app should not ask for permissions on startup. The prompt must only appear when the specific feature is triggered. | Android 14 / iOS 17; Production Build |
| Access Denial Graceful Handling | App requires Location services for a "Find Near Me" feature. | 1. Trigger the feature.<br>2. When the OS prompt appears, select "Don't Allow."<br>3. Attempt to use the feature again. | The app must not crash. It should provide a clear explanation of why the feature is limited and offer a way to proceed without the data. | Android 14 / iOS 17; QA Sandbox |
| Verification of "One-Time" Access Persistence | OS supports "Only this time" or "While using the app" permissions. | 1. Grant "Only this time" access to the Microphone.<br>2. Complete the action and close the app (kill process).<br>3. Relaunch the app and attempt the same action. | The app should prompt for permission again. Access must not be "silently" retained after the session ends. | iOS 17 / Android 14; Production Build |
| Audit of Unnecessary Background Permissions | App is running in the background. | 1. Grant "While using the app" location access.<br>2. Move app to background.<br>3. Monitor system logs/privacy indicators for unauthorized background data polling. | No data (Location, Camera, Mic) should be accessed while the app is in the background unless it is a core, documented background service. | physical Device; ADB/Xcode Console logs |
| Verification of Scoped Storage/Photo Picker | App needs to upload a single image. | 1. Click "Upload Image".<br>2. Observe if the app requests access to the entire gallery or uses the System Photo Picker. | The app should use the system picker to select a specific file rather than requesting broad READ_EXTERNAL_STORAGE permissions. | Android 13+ (Scoped Storage); Emulators/Physical |
| Privilege Escalation via Deep Links | App is logged in with a Standard User role. | 1. Identify a URL/Deep Link intended for Admin users (e.g., /admin/settings).<br>2. Trigger the link from an external source (browser/notes). | The app should intercept the link, validate the user's current scope, and deny access/redirect to the home screen. | iOS/Android; Staging Environment |
