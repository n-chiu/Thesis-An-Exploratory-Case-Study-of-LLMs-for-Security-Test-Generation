# ID 14

## Test Environment Specification
- **Log Source:** Application server or Cloud management console  
- **SIEM Integration:** Splunk, Sentinel, or a local Syslog server  
- **Data Masking:** Hashing/Anonymization service enabled  

| Test Case Title/Scenario | Preconditions | Test Steps | Expected Results |
|--------------------------|---------------|------------|------------------|
| TC-01: Successful Log Capture of User Access | 1. User account exists.<br>2. SIEM listener is active. | 1. Log in with a standard user account.<br>2. Navigate to different modules.<br>3. Log out. | The SIEM should receive logs containing: Timestamp, User_ID, Action: Login/Logout, and IP_Address. |
| TC-02: Administrative Privilege Escalation Logging | 1. Admin account active.<br>2. Audit logging enabled. | 1. Admin logs in.<br>2. Admin changes another user’s permissions.<br>3. Admin creates a new API key. | Logs must explicitly identify the Admin_ID, the Target_User, and the Permission_Change event in the standard format. |
| TC-03: PII and Password Masking Verification | 1. System configured for log scrubbing.<br>2. User login flow active. | 1. Attempt a login with a user ID and password.<br>2. Change user profile details (Email, Phone).<br>3. Check raw logs. | No cleartext passwords or PII (Phone/Email) should appear. These fields should be empty, masked (e.g., ***), or hashed. |
| TC-04: Log Format Standardization (JSON/Syslog) | 1. SIEM configured for specific schema. | 1. Perform any logged action.<br>2. Export the log file or view in SIEM. | The log structure must match the agreed schema (e.g., JSON keys: event_type, severity, source, details). |
| TC-05: Log Generation for Failed Access Attempts | 1. Known user account exists. | 1. Enter an incorrect password 3 times.<br>2. Check SIEM for "Failed Login" alerts. | A "Security Event" log is generated with Severity: High and the source IP, identifying a potential brute force attempt. |
| TC-06: System Performance Under High Log Volume | 1. Load testing tool active.<br>2. SIEM connected. | 1. Simulate 1,000 concurrent user logins.<br>2. Monitor log delivery latency. | Logs are delivered within the specified SLA (e.g., < 5 seconds) without causing application performance degradation. |
