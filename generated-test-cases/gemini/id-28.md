| Valid User Authentication | User is registered in the system with valid credentials. | 1. Send POST request to /login with valid username and password. | Status code 200 OK. Response contains a valid auth token (JWT/Session ID). | QA/Staging Environment; HTTPS enabled. |
| Invalid Credential Rejection | User exists, but incorrect password is used. | 1. Send POST request to /login with valid username but incorrect password. | Status code 401 Unauthorized. No session token provided; error message "Invalid credentials." | QA/Staging Environment; HTTPS enabled. |
| RBAC - Unauthorized Resource Access | User is logged in with "Standard" role permissions. | 1. Authenticate as Standard User.<br>2. Attempt to access /admin/delete-user endpoint. | Status code 403 Forbidden. Access is denied based on insufficient privileges. | QA/Staging Environment; REST API Client. |
| Session Token Expiration | User has an active session token with a short TTL (Time-to-Live). | 1. Authenticate and receive token.<br>2. Wait for the defined timeout period.<br>3. Attempt to call a protected GET /profile endpoint. | Status code 401 Unauthorized. The expired token is rejected, forcing a re-login. | QA/Staging Environment; API Monitoring tools. |
| Secure Logout & Invalidation | User is currently authenticated. | 1. Send POST request to /logout.<br>2. Attempt to use the previous session token to access /profile. | 1. Logout returns 200 OK.<br>2. Second request returns 401 Unauthorized (token is blacklisted/invalidated). | QA/Staging Environment; Browser/API Client. |
| Brute Force Protection | A valid user account exists. | 1. Submit 5+ incorrect login attempts for the same username in rapid succession. | Account is temporarily locked or CAPTCHA is triggered. Status code 429 Too Many Requests. | QA/Staging Environment; Security testing tools. |
| Session Fixation Prevention | User is at the login page. | 1. Capture the session ID before login.<br>2. Log in with valid credentials.<br>3. Check the session ID after login. | The Session ID after login must be different from the pre-login ID. | QA/Staging Environment; Proxy tool (e.g., Burp Suite). |
