| Test Case Title/Scenario | Preconditions | Test Steps | Expected Results | Environment Specification |
|--------------------------|---------------|------------|------------------|----------------------------|
| TC-01: Resource Exhaustion Isolation (DoS) | 1. Monitoring tools active.<br>2. Component A has defined resource quotas (CPU/RAM). | 1. Trigger a recursive loop or memory leak in Component A.<br>2. Observe the performance of Component B and C. | Component A may crash or throttle, but Components B and C remain responsive with no latency increase. | Staging / Kubernetes Cluster with Resource Quotas enabled. |
| TC-02: Lateral Movement Prevention | 1. SSH/Shell access to Component A.<br>2. Network policies implemented. | 1. Gain shell access to Component A.<br>2. Attempt to ping or port-scan Component B's internal IP.<br>3. Attempt to access Component B’s database. | All connection attempts from Component A to unauthorized internal services are dropped or rejected. | VPC with Micro-segmentation / Service Mesh (Istio/Linkerd). |
| TC-03: Privilege Escalation via Service Account | 1. Component A runs under a restricted Service Account/IAM role. | 1. Compromise Component A.<br>2. Attempt to use Component A’s credentials to delete a storage bucket or modify users. | The action is denied (403 Forbidden) because the component’s identity lacks permissions outside its narrow scope. | Cloud Environment (AWS/Azure/GCP) with Least Privilege IAM. |
| TC-04: Cascading Failure (Circuit Breaker) | 1. Component B depends on Component A.<br>2. Circuit breaker pattern implemented. | 1. Simulate a "malicious hang" or 500 error loop in Component A.<br>2. Send a request to Component B that requires data from A. | Component B trips the circuit breaker and returns a graceful fallback/cached response instead of hanging or crashing. | Integration Testing Environment with Resilience4j or similar. |
| TC-05: Data Access Compartmentalization | 1. Component A and B use separate database schemas/users. | 1. Inject a malicious script into Component A to perform a cross-database query.<br>2. Attempt to read "Component B" tables. | The database engine returns an "Access Denied" error; Component A's credentials cannot see Component B's data. | Production-parity Database (PostgreSQL/MySQL) with RBAC. |
