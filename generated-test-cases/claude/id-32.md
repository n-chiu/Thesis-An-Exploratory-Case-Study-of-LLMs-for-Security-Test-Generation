Apply all dependency updates to the manifest and run the package manager install command.<br>2. Execute the full automated test suite.<br>3. | TC-04 | Validate that the runtime platform (language / framework) meets the minimum supported version | 1. Official end-of-life (EOL) schedule is documented for the platform in use.<br>2. | TC-05 | Ensure dependency version pinning strategy prevents automatic installation of incompatible updates | 1. Perform a clean install using only the lock file (e.g. npm ci, pipenv install --ignore-pipfile).<br>2. Verify installed versions match the lock file exactly (no floating ranges resolved differently).<br>3. Intentionally modify one dependency to introduce a minor-version bump and re-run install.<br>4. Confirm that the version bump is rejected or flagged without an explicit lock-file update. | 1. Clean install reproduces exactly the versions declared in the lock file.<br>2. Unauthorized version changes cause the CI pipeline to fail or raise an error.<br>3. Lock file remains the single source of truth for resolved versions. | CI: GitHub Actions / Jenkins. Package managers: npm / pip / yarn. OS: Linux (Ubuntu 22.04). |
