# (32) Software Supply Chain Failures

## Title: Up to date components

## ID: 32

## User Story:
As a Software company Customer, I want all components of the application to have up-to-date libraries and platforms.

## Acceptance Criteria:

All components should be up to date with proper security configuration(s) and version(s).

Verify that third party components come from trusted repositories.

## Edge Cases:

Zero-day vulnerabilities in “latest” versions.

Breaking changes when upgrading libraries.

Transitive dependencies introducing vulnerabilities.

Abandoned or unmaintained libraries.

Different environments running different versions.

Fake or malicious packages from public repositories.

## Constraints:

Dependency scanning (SCA tools).

Use trusted repositories only.

Pin and verify dependency versions (e.g. lock files).

Patching and update cycles.

Test updates in staging before production rollout.

Monitor vulnerability databases (e.g. CVE feeds).

Remove unused dependencies.

Verify package integrity (checksum/signatures).

## Vulnerability Category:

Software Supply Chain Failures

## CWEs:

Software Supply Chain Failures: 1104, 1329, 1357, 1395.
