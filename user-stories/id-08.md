# (8) Authentication

## Title: Session timeout and end after logout

## ID: 08

## User Story:
As a Software company Customer, I want a user logout feature. I need user sessions inactivated after logout and timed out after inactivity or a time limit set by an administrator.

## Acceptance Criteria:

Sessions are invalidated when the user logs out.

Verify that sessions timeout after a specified period of inactivity or after an administratively- configurable maximum time period regardless of activity (an absolute timeout)

## Edge Cases:

User logs out but session remains valid on another tab/device.

Idle timeout not triggered due to background activity (e.g. polling).

Long-running operations interrupted by timeout.

Session tokens reused after logout.

Mobile apps not syncing logout state.

## Constraints:

Server-side session invalidation required on logout.

Idle timeout and absolute timeout must both be enforced.

Timeout values configurable by administrators.

Tokens must be revoked immediately on logout.

Session state must be consistent across distributed systems.

## Vulnerability Category:

Authentication Failures

## CWEs:

Authentication Failures: 384, 613
