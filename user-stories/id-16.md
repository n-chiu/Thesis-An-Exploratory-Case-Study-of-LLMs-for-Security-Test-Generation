# (16) Security Logging and Alerting Failures

## Title: Detection and alert when unauthorized access attempts are made.

## ID: 16

## User Story:
As a Software company Customer, I want the application to detect and alert when there are unauthorized attempts to access my data, login or make changes.

## Acceptance Criteria:

The ability to limit the number of login attempts or send alerts when thresholds have been exceeded.

Ability to send alerts when a role attempts an action beyond privilege level or if attempts to exceed privilege level hit a threshold.

## Edge Cases:

Brute force attacks distributed across IPs (low-and-slow attacks).

Legitimate users triggering alerts due to unusual behaviour (false positives).

Alert fatigue causing important alerts to be ignored.

Attackers testing privilege escalation just below alert thresholds.

Delayed alert delivery.

Disabled or misconfigured alerting systems.

## Constraints:

Rate limiting must be enforced on login and sensitive actions.

Thresholds must be configurable and adaptive.

Alerts must be near real-time.

Integration with SIEM/SOC tools required.

Alerts must include sufficient context for investigation.

Must support anomaly detection (behaviour-based, not just thresholds).

## Vulnerability Category:

Security Logging and Alerting Failures

## CWEs:

Security Logging and Alerting Failures: 221, 223, 778.
