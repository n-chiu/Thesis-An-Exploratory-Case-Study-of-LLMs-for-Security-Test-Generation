# Injection

## Title: Validate input

## ID: 11

## User Story:
As a Software company Customer, I want the application to validate input to be correct and fit for the intended purpose.

## Acceptance Criteria:

Validate all data input from an external entity or client.

## Edge Cases:

Injection attacks.

Unexpected input types.

Extremely large payloads.

File uploads with malicious content.

## Constraints:

Validate on both client and server side (server is authoritative).

Use an allowlist.

Enforce strict data types, formats, and lengths.

Sanitize output to prevent XSS.

Use parameterized queries for database access.

Limit payload sizes.

Centralized validation logic to ensure consistency.

## Vulnerability Category:

Injection  

## CWEs:

Injection: 77, 78, 79, 80, 83, 86, 89, 91, 93
