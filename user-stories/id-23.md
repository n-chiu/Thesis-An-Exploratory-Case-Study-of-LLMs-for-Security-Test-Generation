# (23) Security Misconfiguration + Software or Data Integrity Failures

## Title: Same-Origin Policy

## ID: 23

## User Story:
As a Software company Customer, I want the application to enforce the Same-Origin Policy for content.

## Acceptance Criteria:

Verify that URL redirects and forwards only allow whitelisted destinations, or show a warning when redirecting to potentially untrusted content.

The web or application server is configured by default to deny access to remote resources or systems outside the web or application server.

Implement a Content Security Policy, where possible.

## Edge Cases:

Open redirect vulnerabilities via query parameters.

Misconfigured CORS allowing * with credentials.

Third-party scripts introducing cross-origin risks.

JSONP endpoints bypassing SOP.

Mixed content (HTTP resources on HTTPS pages).

Clickjacking via iframes from malicious domains.

CSP misconfiguration breaking legitimate functionality.

## Constraints:

Enforce strict allowlist for redirects and forwards.

Disable or tightly restrict CORS (no wildcard with credentials).

Implement strong Content Security Policy (CSP).

Use X-Frame-Options or frame-ancestors in CSP.

Block mixed content (HTTPS only).

Validate all external resource inclusions.

Default-deny outbound connections from server where possible.
