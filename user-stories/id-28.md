# (28) Broken Access Control + Authentication Failures

## Title: Authentication, authorization and session management

## ID: 28

## User Story:
As a Software company Customer, I want the web service to implement proper authentication, authorization and session management.

## Acceptance Criteria:

Access to administration and management functions within the Web Service Application is limited to web service administrators.

Verify the use of session-based authentication and authorization.

Avoid the use of static "API keys" and similar.

## Edge Cases:

Admin endpoints exposed without proper authorization checks.

Session tokens reused across users or services.

Long-lived sessions increasing attack window.

API keys hardcoded in clients or leaked in repos.

Mixed authentication methods causing inconsistencies.

Privilege escalation via misconfigured roles.

## Constraints:

Strong authentication.

Role-based or attribute-based access control enforced.

Session tokens must be short-lived and securely stored.

No static API keys; use rotating or dynamic tokens.

Admin endpoints must be strictly restricted and audited.

Consistent auth enforcement across all endpoints

Secure session storage and validation required.

## Vulnerability Category:

Broken Access Control + Authentication Failures

## CWEs:

Broken Access Control: 276, 281, 282, 283, 732, 863
